1，简介

js的数据类型可以分为两个大类

  • 简单类型（基本类型)

    • Number :整数或者小数
    • String ： 文本
    • Boolean ：true和false两个特殊值
    • undefined ：未定义的值
    • null ：空值
    • symbol ：

  • 复杂类型（合成类型）

    • Object：各种值得集合
      • 狭义的对象
      • 数组
      • 函数

2，typeof

typeof的作用是返回一个值的数据类型

除此之外还有一个运算符和一个方法可以确定一个值的数据类型。

  • typeof对于各种数据类型的返回值

    typeof 1          //number
typeof "aa"       //string
typeof true       //boolean
typeof undefined  //undefined
function a(){}
typeof a          //function
typeof null       //object
typeof {}         //object
typeof []         //object

3，null和undefined

3.1 概述

在大多数的编程语言中并没有使用两个值来表示空这一概念。在设计之初，Eich其实只涉及了null，但他后来觉得不妥，就又加了一个undefined。

将一个变量赋值为undefined或是null的语法效果基本没有任何区别。

null == undefined     //true
null === undefinded   //false

3.2 区别

  • null表示一个空的对象，当把它转换为数值的时候可以自动转变成0,
    undefined则不行。

    Number(null); // 0
    Number(undefined) //NaN

  • 使用的场景不同

    • undefined
    1,声明变量但是没有赋值

      var test;                   //undefined

  2,调用函数时，应该提供的参数没有被提供

      function test1(a){
        return a;
      }

      test1();                    //undefined

  3,对象没有赋值的属性

      window.aaaaa;               //undefined

  4,没有设置返回值的函数，默认返回值就是undefined


      function test4(){

      }

      test4();                    //undefined





* `null`

  `null`表示空值，即该处的值现在为空。调用函数时，某个参数未设置任何值，这时就可以传入`null`，表示该参数为空。比如，某个函数接受引擎抛出的错误作为参数，如果运行过程中未出错，那么这个参数就会传入`null`，表示未发生错误。

4，Number

4.1 整数和浮点数

与c语言不同，js所有的数字都是用64位浮点数来表示的。但是有些运算只有整数才能完成，此时js会将64位浮点数，转换成32位整数。

  • 由于计算机对于浮点数的存储机制，决定了浮点数结算结果有时不会是特别精确的值。
0.1 + 0.2 === 0.3         //false


0.7 / 0.1                 //0.999999999999999


(0.6-0.3) === (0.7-0.4)   //false

  • js可以精确表示-2^53^ 到 2^53^ ,当大于这个数的时候就无法保持精度，这个数是16位，即js可以保证15位数的计算精度。

    Math.pow(2,53)     //9007199254740992
Math.pow(2,53)+1   //9007199254740992

  • js可以表示2^1024^ 到 2^-1023^

    • 大与1024次方发生”正向溢出”
Math.pow(2,1025)        //Infinity

  • 小于-1075(指数部分-1023和小数部分的52位)次方发生”负向溢出”

    Math.pow(2,-1075)      //0

  • Number对象的MAX_VALUE和MIN_VALUE分别返回js所能表示的最大的数

    Number.MAX_VALUE      // 1.7976931348623157e+308
Number.MIN_VALUE      // 5e-324

4.2数值的表示法

  • 科学计数法，一个整数 + e +指数部分

    111e10              //1110000000000

  • 有两种情况会让js自动转为科学计数法

    • 小数点前多与21位

      1231131231231231231234   //1.2311312312312311e+21
* 小数点后的0多与5个


      0.0000001               //1e-7

4.3数值的进制

对于各种进制在js的写法

  • 2进制 加前缀0b或者0B的数值

  • 8进制 加前缀0O或者0o的数值

  • 16进制 加前缀0x或者0X的数值

  • PS*:如果在一个数值前加0并且这个数的每一位都小于8，name也可以用来表示8进制，否则就是十进制。

当相应进制中出现了不属于该进制的数值，那么便会报错

4.4 特殊值

1) 0
`+0`和`-0`，js里面的0有两个，它们完全等价，几乎所有场合他们都会被当做一个普通的`0`来使用，唯一的区别是在当**分母**的的时候。

    0 === +0     //true
    0 === -0     //true
    +0 === -0    //true


    1/0        //Infinity
    1/+0       //Infinity
    1/-0       //-Infinity

#####2) NaN

(1)含义
它主要出现在将字符串解析成数字出错的场合

  1-'e'              //NaN    


它本身类型属于`Number`

  typeof NaN         //Number


一些计算出现问题也会返回NaN

  Math.acos(2)       // NaN
  Math.log(-1)       // NaN
  Math.sqrt(-1)      // NaN
  0/0                //NaN
（2）运算规则

它不等于任何值，包括它本身

NaN === NaN       //false

与任何数运算得到NaN

1+NaN             //NaN
(3)isNaN
3) Infinity

Infinity大于一切数值，-infinity小于一切数值,但是两者与NaN比较的结果都是false

 Infinity > NaN    //false
-Infinity > NaN    //false
 Infinity < NaN    //false
-Infinity < NaN    //false

5，字符串

用’’或””或``包起来的一个或者多个字符就是字符串。

5.1换行书写

1）使用+
var str = "a"
          +"b";      

a             //ab
2) `
var str  = `aaa
aaa
aaa`;

str                 //aaaaaaaaa
3) \

这种方式很容易出现问题，当反斜杠后面有任何内容，包括空格，都会导致报错。

var str = "aaa\
          aaa\
          aaa";


    str               //aaaaaaaaa

5.2转义字符\

1）一些常用的转义字符
\0 ：null（\u0000）
\b ：后退键（\u0008）
\f ：换页符（\u000C）
\n ：换行符（\u000A）
\r ：回车键（\u000D）
\t ：制表符（\u0009）
\v ：垂直制表符（\u000B）
\' ：单引号（\u0027）
\" ：双引号（\u0022）
\\ ：反斜杠（\u005C）
2）三种特殊用法

  • \HHH

    反斜杠后面紧跟三个八进制数（000到377），代表一个字符。HHH对应该字符的 Unicode 码点，比如\251表示版权符号。显然，这种方法只能输出256种字符。

  • \xHH

    \x后面紧跟两个十六进制数（00到FF），代表一个字符。HH对应该字符的 Unicode 码点，比如\xA9表示版权符号。这种方法也只能输出256种字符。

  • \uXXXX

    \u后面紧跟四个十六进制数（0000到FFFF），代表一个字符。XXXX对应该字符的 Unicode 码点，比如\u00A9表示版权符号。

5.3 字符串与数组

可以通过数组来使用显示字符串，但是并不可以被修改。

var test = "Hello";
test[0]         //H

test[8]         //undefined

test[0] = "h";
test;           //Hello

5.4 length属性

返回字符串长度，但是也没办法改变。

var test = “hello”;

test.length; //5

test.length = 1;
test.length; //5

5.5 Base64转码

主要作用是，将任意值转成 0～9、A～Z、a-z、+和/这64个字符组成的可打印字符。

1）两个原生方法

  • btoa()将任意值转变成Base64

  • atob()将Base64转换成原来的值

#####2）转换字符

这个方法对于ASCII码的字符支持很差，需要加一个中间环节

  function b64Encode(str) {

    return btoa(encodeURIComponent(str));

  }

function b64Decode(str) {

    return decodeURIComponent(atob(str));

  }

b64Encode('你好')                        // "JUU0JUJEJUEwJUU1JUE1JUJE"
b64Decode('JUU0JUJEJUEwJUU1JUE1JUJE')   // "你好"

6，对象

对象是一种十分特别的数据类型，是由各种原始类型组合而成的“键值对”集合。

6.1键名

对象的所有键名都是字符串，在定义的时候键名会被自动转换成字符串，数值也会被转换成字符串

var obj = {
  a:1,
  b:2

}

等价于

var obj = {

  'a':1,
  'b':2

}

6.2 引用

这个和c语言的指针真的很像，都是存的是内存地址，如果不同的变量名指向同一个对象，那么它们都是这个对象的引用，也就是说指向同一个内存地址。修改其中一个变量，会影响到其他所有变量。这是对象与原始类型的最大的区别。

var obj = {hehe:1};
var n = 1;

function a(test){

    test.hehe = 2;



}

a(obj);        //这里传入的是这个对象
               //这样test和obj所引用的内存地址完全相同
               //这样修改其中一个就会影响到其他的
a(n);
console.log(obj.hehe);
console.log(n)

6.3 属性

1）.和[]

个人认为.这种方法看似简单好用，但实际上功能不是很全面。因为如果想要通过字符串变量来使用属性，就只能使用[],还有当键名是数字时也没办法使用.。

  var obj = {

    name:'zjy',
    gender:'female',
    1:666
  }


var a = name;
var b = 1;

obj.a;                //undefined
obj.name              //zjy
obj.1                 //报错啦Uncaught SyntaxError: Unexpected number
                      // 未捕获的语法错误：意料之外的数字
obj.b                 //undefined

obj['a']              //undefined
obj['name']           //zjy
obj[a]                //zjy
obj[1]                //666
obj['1']              //666
obj[b]                //666
2)查看属性

Object.keys()方法

  var obj = {
  key1: 1,
  key2: 2
};

Object.keys(obj);       //['key1', 'key2']

#####3) 删除属性
delete命令用于删除对象的属性，删除成功后返回true。不管删除的属性是否存在都会返回true

var obj = {

  a:1,
  b:2

}

Object.keys(obj)    //['a','b']

delete obj.a        //true

Object.keys(obj)    //['b']
4) in
不管是不是继承的属性都会返回ture

    var obj = {

        a:1,
        b:2

    }

   'a' in obj;           // true
   'toString' in obj     //true

  • 判断是否是自身的属性 hasOwnProperty

    var obj = {};
if ('toString' in obj) {
  console.log(obj.hasOwnProperty('toString')) // false
}
5) 遍历属性：for…in
  • 它遍历的是对象所有可遍历（enumerable）的属性，会跳过不可遍历的属性。
  • 它不仅遍历对象自身的属性，还遍历继承的属性。
  • 可以设置属性的可遍历性，有些继承属性并不会被遍历到就是这个原因。
var obj = {

  a:1,
  b:2

}

obj.toString();         //"[object Object]"

for(var i in obj){

  console.log(i);

}                       // a b
  • 也可以使用hasOwnProperty判断是不是自身属性。
6）with

操作同一个对象的多个属性 

var obj = {
  p1: 1,
  p2: 2,
};
with (obj) {
  p1 = 4;
  p2 = 5;
}

这个属性尽量不要使用，当操作obj中不存在的属性时，传建一个全局变量，不建议使用

7，类型的转换

7.1转成Sstring
1) window.string()
window.String(1)                 //"1"
window.String(ture)              //"true"
window.String(undefined)         //undefined
window.String(({a:1}))           //"[object Object]"
window.String(null)              //"null"
2) xxx.toString()

这个属性很常用但是功能的局限性也很大。

var a = 1;
a.toString();                 //"1"

a = true;
a.toString();                 //"ture"

a = false;
a.toString();                 //"false"

a = null;
a.toString();                 //Uncaught TypeError: Cannot read property 'toString' of null

a = undefined;
a.toString();                 //Uncaught TypeError: Cannot read property 'toString' of undefined

a = {aa:1};
a.toString()                  //"[object Object]"
3) +""

比较常用并且效果很好，加号的性质是，如果两个不同数据类型的数据相加，会优先转换成字符串

1 + "1"                //"11"

1+"1"+true             //"11true"

1+true+1               //"21"

将想要转换成字符串的数据加一个””(是空字符串，不是空格字符串)

1 + ""                //"1"
true + ""             //"true"
false + ""            //"false"
null + ""             //"null"
undefined+ ""         //"undefined"

8，深拷贝vs浅拷贝

基本类型的简单赋值就是深拷贝,但是一般不会考虑基本类型。只会考虑对象的深拷贝。

var a = 1;
var b = a;
b = 3;
console.log(a);       //1
console.log(b);       //3

对象默认的赋值方式就是浅拷贝

var a = {test :1};
var b = a ;
b.test = 2;
console.log(a.test);  //2
console.log(b.test);  //2
