1，什么是脚本？

按照一定的语法，告诉计算机你想让它做什么。

2，写脚本

2.1脚本功能

1，运行 ./demo2.sh xxx 之后，将会创建一个 xxx 目录

2，xxx 目录里面应该有一个 index.html 文件，一个 css 目录 和 一个 js 目录，也就是

xxx
|-- css
|   |-- style.css
|-- index.html
|-- js
    |-- index.js

3，其中 index.html 文件里面的内容应该为 hi html

4，其中 css/style.css 文件里面的内容应该为 hi css

5，其中 js/index.js 文件里面的内容应该为 hi js

2.2 步骤

  • 建立目录

    建立一个专门存放脚本的文件夹，这里创建了local作为存放脚本的目录，并进入目录

    mkdir local
cd ~/local

  • 创建文件

    脚本文件后缀一般是sh,但是txt也可以执行

    touch demo.txt

  • 编辑文本内容

    mkdir $1                          //根据参数创建一个目录
cd  $1                            //进入创建好的目录
mkdir css js                      //分别创建css和js两个目录
touch index.html css/style.css js/main.js
                                  //创建三个空文件 
echo "hi html" > index.html       //将"hi html"写入index.html
echo "hi css" > css/style.css     //将"hi css"写入style.css
echo "hi js" > js/main.js         //将"hi js"写入main.js
exit                              //类似于c语言里面的return 0，是一种好习惯

  • 执行权限

    现在还不能愉快地执行脚本，因为当ls -l后会发现现在这个文件并没有执行权限，

    chmod +x demo.txt

  • 执行脚本

    现在终于可以在任何位置愉快地执行脚本了,输入如下指令就可以建立一个叫xxxx的文件夹，里面有我们想要的一切~

    sh ~/local/demo.txt xxxx

  • 设置PATH

    说实话，现在脚本虽然可以正确的运行了，但是还并不完美。因为想要执行它，还要输入一大串的东西，太麻烦了，可不可以简化它，就好像ls、cd这些内置命令一样可以简短一点。因为本质上，大家都是脚本，凭什么我们不如人家好用呢？？？
    答案是肯定的，我们当然也能变得这么好用。

    方法只有一个将我们的脚本也加入PATH目录

    在文件.bashrc中加入

    export PATH="local的绝对路径:$PATH"

    运行

    source ~/.bashrc

    现在输入demo.txt xxx就可以运行了，还可以将后缀去掉

    mv ~/local/demo.txt ~/local/demo
