1，前言

现在iframe已经很少用了，这是一个可替换元素。这个元素的属性很多，但是并不常用，以后用到时候在具体去了解，下面是比较常用的属性。

2，iframe

  • frameborder

    默认有外边框

    <iframe src = "http://qq.com" ><iframe>

    去掉外边框

   <iframe frameborder = "0" src = "http://qq.com" ><iframe>

![noborder](/zjyshy.github.io/img/noborder.png)

  • name

    属性需要搭配a标签使用,这样a标签href的地址，就会从name为xxx的iframe中打开

    <iframe name = xxx frameborder = "0" src = "http://qq.com" ><iframe>
<a target="xxx" href="http://www.baidu.com">xxx </a>

  • src

    支持相对路径

    <iframe src = "./index2.html" ><iframe>

![相对路径](/zjyshy.github.io/img/index2.png)

    //表示使用当前页面所使用的协议

    <iframe src = "//qq.com" ><iframe>

四种target值

想要更好的理解这四个值，还需要结合iframe。

.

├── index.html

├── index2.html

└── index3.html

index3.html的内容

<a target="_blank" href="http://www.baidu.com">_blank </a>
<a target="_top" href="http://www.baidu.com">_top</a>
<a target="_self" href="http://www.baidu.com">_self </a>
<a target="_parent" href="http://www.baidu.com">_parent </a>

index2.html的内容

<iframe src="./index3.html" frameborder="0"></iframe>

index.html的内容

<iframe  src="./index2.html" frameborder="0"></iframe>

_blank 会打开一个新的窗口

_top 会在index.html这个祖宗页面打开

_self 会在ifrem中打开的index3.html中打开

_parent会在ifrem中打开的index2.html中打开

download属性
  • content-type:application/octet-stream是通过设置http内容让页面变成下载

  • a标签也可以通过设置download来实现下载页面的功能

    下载这个页面

    <a class = "downloadResume" href="#" download>下载这个页面</a>

src和href

  • 在HTML中有的标签使用src有的则是href甚至还有url这里着重说一下href和src的区别。

  • src表示引入 （img、script、iframe），是页面必不可少的内容，在请求src资源时会将其指向的资源下载并应用到文档内。

    • href表示引用（link、a），建立和当前元素（锚点）或当前文档（链接）之间的链接
